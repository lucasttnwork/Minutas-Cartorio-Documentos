INSTRUCOES ESPECIFICAS:

1. LIVRO/FOLHA/TERMO - ATENCAO MAXIMA:
   - Extraia APENAS os numeros que aparecem no documento
   - Formato correto: "B-0008", "030", "1370"
   - Formato ERRADO: "ffff(0003)", "ggg (050)" - NUNCA use isso
   - Se nao conseguir ler, use null

2. AVERBACOES - CAPTURE TODAS:
   - Procure na secao "AVERBACOES/ANOTACOES" e no verso do documento
   - Separacao judicial: identifique data, processo, vara
   - Divorcio: identifique data do transito em julgado, processo
   - Conversao de uniao estavel: capture a base legal (ex: Art. 8 Lei 9278/96)
   - Alteracao de nome: indique nome anterior e novo nome

3. REGIME DE BENS E PACTO ANTENUPCIAL:
   - Identifique o regime exato
   - Se mencionar "ESCRITURA DE PACTO ANTENUPCIAL", extraia: cartorio, livro, folhas, data
   - Comunhao Universal COM pacto e uma situacao comum

4. NOMES DOS CONJUGES:
   - nome_completo: nome como aparece no cabecalho/identificacao principal
   - nome_solteiro: nome ANTES do casamento (geralmente em averbacoes de separacao/divorcio)
   - nome_casado: nome APOS o casamento (se houve alteracao)
   - Se manteve o mesmo nome, nome_solteiro = null e houve_alteracao_nome = false

5. SITUACAO ATUAL DO VINCULO:
   - CASADOS: se nao ha averbacao de separacao ou divorcio
   - SEPARADOS: se ha averbacao de separacao mas nao de divorcio
   - DIVORCIADOS: se ha averbacao de divorcio
   - VIUVO(A): se ha averbacao de obito de um dos conjuges

6. CAMPOS NULL:
   - Use null para qualquer campo que NAO esteja visivel no documento
   - NAO invente informacoes
   - NAO use "Informacao Ausente" - use null

VALIDACOES AUTOMATICAS (o modelo deve verificar):
- CPF: formato XXX.XXX.XXX-XX
- Datas: formato DD/MM/AAAA
- Data de nascimento deve ser anterior a data de casamento
- Data de emissao deve ser posterior a data de casamento
