# PROMPT PARA ANÁLISE DE MATRÍCULA DE IMÓVEL - v2.0 CORRIGIDO

## REGRAS OBRIGATÓRIAS (LEIA PRIMEIRO!)

1. **CADEIA DOMINIAL COMPLETA**: Você DEVE listar TODOS os proprietários desde a abertura da matrícula até hoje. NUNCA omita o proprietário original.

2. **ÔNUS COMPLETOS**: Capture TANTO ônus ativos QUANTO históricos (cancelados). Use campos separados: `onus_ativos` e `onus_historicos`.

3. **VERIFICAR CANCELAMENTOS**: Procure SEMPRE por termos como "CANCELADA", "QUITADA", "BAIXADA" nas averbações. Um ônus cancelado NÃO deve aparecer em `onus_ativos`.

4. **NUNCA CONFUNDIR**:
   - OFICIAL DO CARTÓRIO (oficiais, escreventes, etc.) ≠ PARTE DO NEGÓCIO (vendedor, comprador)
   - O oficial CERTIFICA o documento, ele NÃO É vendedor/comprador

5. **NUNCA INVENTAR DADOS**: Se algo está ilegível ou ausente, retorne `null`. NUNCA invente datas, valores ou nomes.

6. **EXPLICAÇÃO CONTEXTUAL OBRIGATÓRIA**: O campo `explicacao_contextual` DEVE conter 3-5 parágrafos explicando a história do imóvel. NUNCA deixe vazio ou com apenas "#".

---

## CONTEXTO DO DOCUMENTO

Uma **matrícula de imóvel** é o documento oficial que registra toda a história de um imóvel no Brasil. Ela contém:
- **Descrição do imóvel**: endereço, áreas, confrontações
- **Proprietários originais**: quem abriu a matrícula
- **Registros (R-1, R-2, R-3...)**: transmissões de propriedade (vendas, doações, heranças) e ônus (hipotecas, penhoras)
- **Averbações (AV-1, AV-2...)**: modificações, cancelamentos, alterações de estado civil

**O que são R-X e AV-X?**
- **R-1, R-2, R-3...** = REGISTROS. Podem ser:
  - Transmissão de propriedade (venda, doação, herança)
  - Constituição de ônus (hipoteca, alienação fiduciária, penhora)
- **AV-1, AV-2...** = AVERBAÇÕES. Podem ser:
  - Cancelamento de ônus (hipoteca quitada)
  - Mudança de estado civil
  - Alteração de denominação social de empresas
  - Retificação de dados

## TAREFAS OBRIGATÓRIAS

### 1. REESCRITA INTERPRETADA
Transcreva o documento COMPLETO, organizando por seções:
- Dados de identificação (matrícula, cartório, livro, ficha)
- Descrição do imóvel
- Proprietário original (da abertura da matrícula)
- Todos os registros R-X em ordem
- Todas as averbações AV-X em ordem
- Marque claramente se há texto como "(continua no verso)"

### 2. ANÁLISE DA CADEIA DOMINIAL (CRÍTICO!)

**PASSO A PASSO OBRIGATÓRIO:**

a) **Identifique o PROPRIETÁRIO ORIGINAL**:
   - Está no início da matrícula, após a descrição do imóvel
   - Geralmente começa com "Proprietário:" ou logo após a descrição
   - Inclui o TÍTULO AQUISITIVO (ex: "conforme Transcrição nº 00.000")

b) **Siga CADA registro R-X cronologicamente**:
   - R-1: Quem transmitiu? Para quem? Que tipo de ato? (venda, doação, hipoteca?)
   - R-2: Mesmo processo
   - Continue até o último registro

c) **Verifique averbações de cancelamento**:
   - Procure por "CANCELADA", "QUITADA", "BAIXADA"
   - Se encontrar, o ônus correspondente está EXTINTO

d) **Determine os PROPRIETÁRIOS ATUAIS**:
   - São os ÚLTIMOS ADQUIRENTES de uma transmissão de propriedade (não de ônus)
   - Se não houve nenhuma transmissão após a abertura, é o proprietário original

### 3. ANÁLISE DE ÔNUS E GRAVAMES

**Tipos de ônus comuns:**
- HIPOTECA: Garantia de empréstimo
- ALIENAÇÃO FIDUCIÁRIA: O credor é "dono" até quitação
- PENHORA: Bloqueio judicial
- USUFRUTO: Direito de uso por terceiro
- SALDO DEVEDOR: Parcelamento ao vendedor

**Para CADA ônus identificado:**
1. Verifique se há averbação de cancelamento posterior
2. Se CANCELADO → vai para `onus_historicos`
3. Se ATIVO → vai para `onus_ativos`
4. Se documento truncado ("continua no verso") → marque status como "DESCONHECIDO"

### 4. CATALOGAÇÃO DE DADOS

Extraia TODOS os dados estruturados conforme schema abaixo.

## INSTRUÇÕES ESPECÍFICAS POR SITUAÇÃO

### Quando o documento indica "continua no verso":
```json
{
  "contexto_pagina": {
    "documento_completo": false,
    "alertas_pagina": ["Documento indica continuação no verso - informações podem estar incompletas"]
  },
  "alertas": [
    {
      "gravidade": "ALTA",
      "tipo": "DOCUMENTO_INCOMPLETO",
      "mensagem": "R-2 incompleto - documento indica continuação no verso. Solicitar página complementar."
    }
  ]
}
```

### Quando NÃO há transmissão de propriedade (só a abertura):
O proprietário original É o proprietário atual.

### Quando há múltiplas transmissões (R-1, R-3, R-5...):
Siga a cadeia até o ÚLTIMO registro de transmissão de propriedade (não de ônus).

### Quando o ônus NÃO tem informação de cancelamento:
Marque como ATIVO (princípio da cautela):
```json
{
  "onus_ativos": [
    {
      "tipo": "HIPOTECA",
      "registro": "R-2/00.000",
      "status": "ATIVO",
      "observacoes": "Sem averbação de cancelamento encontrada no documento"
    }
  ]
}
```

### Como distinguir OFICIAL DO CARTÓRIO de PARTE DO NEGÓCIO:
- **OFICIAIS/ESCREVENTES** (IGNORAR como pessoa_relacionada):
  - Aparecem no FIM do documento, na assinatura/certificação
  - Exemplos: "Nome do Oficial - Oficial", "Escrevente: NOME DO ESCREVENTE"

- **PARTES DO NEGÓCIO** (INCLUIR como pessoa_relacionada):
  - Aparecem NOS REGISTROS (R-X) e AVERBAÇÕES (AV-X)
  - São: vendedores, compradores, doadores, donatários, credores, devedores

### Valores em moedas antigas:
- Cr$ = Cruzeiros (até 1986)
- Cz$ = Cruzados (1986-1989)
- NCz$ = Cruzados Novos (1989-1990)
- Cr$ = Cruzeiros novamente (1990-1993)
- CR$ = Cruzeiros Reais (1993-1994)
- R$ = Reais (1994-atual)

Mantenha o valor ORIGINAL com a moeda da época. Não converta.

Antes de retornar o JSON, verifique:

1. [ ] Campo `cadeia_dominial` contém o proprietário original?
2. [ ] Campo `proprietarios_atuais` está preenchido (não vazio)?
3. [ ] Todos os ônus com "CANCELADA/QUITADA" estão em `onus_historicos` (não em `onus_ativos`)?
4. [ ] Campo `explicacao_contextual` tem 3-5 parágrafos (não está vazio ou com "#")?
5. [ ] O campo `pessoa_relacionada` NÃO contém oficiais do cartório?
6. [ ] Todas as datas são do documento (não inventadas)?
7. [ ] Se documento truncado, há alerta em `alertas`?

## LEMBRETE FINAL

Este prompt foi criado para resolver os seguintes problemas críticos identificados:
1. ❌ Proprietários atuais vazios → ✅ Sempre preencher a partir da cadeia dominial
2. ❌ Ônus ativos vazios quando há ônus → ✅ Verificar cancelamentos antes de classificar
3. ❌ Cadeia dominial incompleta → ✅ Incluir SEMPRE o proprietário original
4. ❌ Oficial do cartório como vendedor → ✅ Distinguir autoridade de parte
5. ❌ Explicação contextual vazia → ✅ Obrigatório 3-5 parágrafos
6. ❌ Ônus históricos não capturados → ✅ Campo `onus_historicos` separado
7. ❌ Matrículas relacionadas ignoradas → ✅ Campo `matriculas_relacionadas`

SIGA ESTAS INSTRUÇÕES COM RIGOR. A precisão jurídica desta análise é CRÍTICA para operações imobiliárias.
